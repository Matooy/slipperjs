Structure

Overview

Basically, component/ directory is free space for developer.
But, template/ and view/ directory is reserved as default. 

▾ component/    <- Your working space.
  ▸ template/   <- Put all template files as .html
  ▾ view/       <- Put all view files as .js
    ▸ @layout   <- This directory is hidden. Unaccessable from Router.
▸ config/       <- Configurations.
▸ core/         <- Core files. Please don't touch, until the time comes!!
▸ i18n/         <- i18n files.
▸ module/       <- Extendable constructors.
▸ vendor/       <- Vendors, external libraries and etc.

Hidden directories

All files within a directory which name is started with "@" will be ignored by Router.
This means that you can hide files from basic page transition.
(Be careful. This is the Client-Side framework. It is difficult to hide all things completely.)
If you want to make partial view files, make a directory like "@~~~" and put files in it.

"@" is predefined as default. But you can change this char at config/main.js.
Please see also Basic: Config page.
