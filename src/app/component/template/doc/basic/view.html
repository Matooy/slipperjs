View

SlipperJS's view is similar to BackboneJS view.
(Also you can extend Backbone View object like View.extend(Backbone.View).)

Properties

template

el

View's element definition.


el: '<div id="sample">'

$el

jQuery Object.

overrides


overrides: {
  template:  "TEMPLATE_PATH",
  cacheable: Boolean
}

Methods

Usages for essential methods.

load_partials ( paths : Object, param : Object )

Load paths and apply 


load_partials({ header: "common/header" });

load_template ( path : String )

This method will start loading a template file with this.template and this.overrides.template. this.overrides.template will always overrides path value.

Hooks

init ( $.Deferred, Parameter )

This will be called when view instance is created. If View.overrides.cacheable is true, cached view's init() won't be called.
Always you have to resolve or reject 1st argument $.Deferred object.

When you resolved $.Deferred and pass some parameter as 1st argument,
This parameter will be pass to render, ready, inserted methods. 


init: function( df : $.Deferred, param : Object ){
  // View initialization success
  return df.resolve(param);
}

ready ( parameter : Object )

Called when render() is called.


ready: function(param){
  // Do something like this.
  // this.$el.find('#something').html(param.sample)
}

inserted ( parameter : Object )

Called when this.$el's DOMNodeInserted events fired.

Action

When hash or query-string changed, action() method will be called.

Query: ?

?query changes trigger do_<#HASH>({ param: query }). If #hash is not set, only action() method will be called.

Hash: #

#hash changes trigger do_<#HASH> action.
For example, URL is changed /sample to /sample#say, do_say action will be called.
